# /finishing-a-development-branch — Branch Completion

**When**: After all tasks in `/subagent-driven-development` are completed and reviewed.

---

## Process

### 1. Verify tests (if test suite exists)

```bash
# Only run if test suite detected
[ -f package.json ] && grep -q '"test"' package.json && npm test
```

- ❌ Tests fail → DO NOT proceed. Report and ask for action.
- ✅ Tests pass (or no test suite) → continue.

### 2. Show commits since main

```bash
git log --oneline main..HEAD
```

### 3. Present 4 options

```
✅ [X tests passing | no test suite]

What do we do with feat/<branch-name>?

[1] Merge local    — merge into main, verify, delete branch + worktree
[2] Push + PR      — push and open Pull Request (worktree preserved)
[3] Keep           — preserve branch and worktree for later
[4] Discard        — delete permanently (requires typing "discard")

Choose: [1/2/3/4]
```

### 4. Execute chosen option

**Option 1 — Merge local**:
```bash
git checkout main
git merge feat/<branch-name>
# run tests again if test suite exists
git branch -d feat/<branch-name>
git worktree remove .worktrees/<branch-name>
```

**Option 2 — Push + PR**:
```bash
git push -u origin feat/<branch-name>
gh pr create --title "<feature name>" --body "<auto-generated summary>"
```
Worktree preserved.

**Option 3 — Keep**: No action. Branch and worktree preserved.

**Option 4 — Discard**: Ask user to type "discard" explicitly, then:
```bash
git worktree remove --force .worktrees/<branch-name>
git branch -D feat/<branch-name>
```

---

## Rules

- ❌ NEVER merge with failing tests
- ❌ NEVER discard without explicit "discard" typed by user
- ✅ Always present all 4 options — never assume preference
- ✅ Worktree cleanup only for options 1 and 4
